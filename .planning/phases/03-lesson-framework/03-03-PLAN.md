Server Action contract (this plan creates it):
  markSubComponentComplete(subComponentId: string): Promise<void>
    1. zod z.string().uuid() validate → throw on invalid
    2. getUser() server-side → redirect('/login') if none (NEVER accept user_id from client)
    3. verify sub_component_id exists in sub_components (select id, lesson_id ... single) → throw if not found
    4. upsert { user_id: user.id, sub_component_id, completed_at } onConflict 'user_id,sub_component_id'
    5. revalidatePath for the lesson route so the Server Component re-fetches ground-truth

Route: /levels/[levelSlug]/lessons/[lessonId]. revalidatePath MUST match this exact route shape.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| student browser → SubComponentList (client) → markSubComponentComplete (Server Action) | Client passes only a sub_component_id; user_id is re-derived server-side via getUser() |
| authenticated role → Postgres (RLS) | Progress writes gated by RLS with check `(select auth.uid()) = user_id`; reads scoped to own rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | markSubComponentComplete user_id | mitigate | user_id derived from getUser() server-side, never from the client; RLS with check rejects mismatches as a second layer. Asserted in actions.test.ts. |
| T-03-10 | Tampering | phantom sub_component_id | mitigate | Server Action verifies the sub_component_id exists in sub_components before upsert; throws 'Sub-component not found' otherwise. |
| T-03-11 | Tampering | input validation | mitigate | zod `z.string().uuid()` validates subComponentId before any DB call (V5 Input Validation). |
| T-03-12 | Denial of Service | double-tap / rapid clicks | mitigate | Idempotent upsert on composite PK (onConflict user_id,sub_component_id) + button disabled once optimistic-complete — no duplicate rows, no error. |
| T-03-13 | Information Disclosure | lesson progress fetch | mitigate | Progress query scoped to `eq('user_id', user.id)` and enforced by RLS SELECT policy; cannot read another student's progress. |
</threat_model>
