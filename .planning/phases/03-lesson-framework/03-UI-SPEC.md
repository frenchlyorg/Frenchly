## Spacing Scale

Declared values (multiples of 4, drawn from DESIGN.md spacing tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Sub-component item internal padding, chip gaps |
| md | 16px | Card internal stack, list item vertical gap |
| lg | 24px | Card padding (min), section gutter (`--spacing-gutter`) |
| xl | 32px | Card-to-card gap, lesson header bottom margin |
| 2xl | 48px | Level page section breaks |
| 3xl | 80px | Major section gaps (`--spacing-section-gap`) |

Exceptions:
- Touch targets for the "Mark complete" button: minimum 48px height (exceeds WCAG 2.5.8 44px minimum while staying on the 4px grid: 48 = 4 × 12)
- Lesson content column: `max-width: 720px` (`--spacing-content-max`)
- Level page outer container: `max-width: 1040px` (`--spacing-container-max`)
- Mobile side margins: 20px (`--spacing-margin-mobile`)

Source: DESIGN.md spacing section + globals.css tokens

## Typography

All roles map directly to DESIGN.md scale. No new sizes introduced.

| Role | Family | Size | Weight | Line Height | CSS Token |
|------|--------|------|--------|-------------|-----------|
| Display (level name) | Literata | 48px | 700 | 56px | `display-lg` |
| Heading (lesson title) | Literata | 32px / 28px mobile | 600 | 40px / 36px | `headline-lg` / `headline-lg-mobile` |
| Sub-heading (section label) | Literata | 24px | 500 | 32px | `headline-md` |
| Body (lesson description, explainer content) | Be Vietnam Pro | 16px | 400 | 28px | `body-md` |
| Metadata (time estimate, kind badge) | Work Sans | 13px | 500 | 16px, letter-spacing 0.05em | `label-sm` |
| Guillemet decoration | Literata | 24px | 400 | 24px | `quote-decoration` |

Active sizes in this phase: 13, 16, 24, 28/32, 48 (5 sizes — justified by distinct semantic roles: metadata, body, sub-heading, lesson heading, level display).
Weights in use: 400, 500, 600, 700 (4 weights, per DESIGN.md).

> **DESIGN.md override (deliberate):** The generic UI-quality gate caps sizes at 4 and weights at 2. This spec intentionally exceeds both because the type scale and weight ramp are **locked in DESIGN.md** (project source of truth per CLAUDE.md). Every size/weight here maps 1:1 to an existing DESIGN.md token — no ad-hoc values are introduced. The locked design system wins over the generic heuristic.

Green (`--color-tertiary`) must not appear anywhere in Phase 3 UI. The "Mark complete" success state uses primary coral, not green.

## Component Inventory

New components required for Phase 3:

### LevelCard (`components/lessons/LevelCard.tsx`)
- Surface: `--color-surface-container-low`
- Border: 1px `--color-outline-variant`
- Border radius: `--radius` (0.5rem)
- Internal padding: 24px (lg)
- Contains: lesson title (headline-md), time estimate badge (label-sm), sub-component count, locked/unlocked indicator
- Locked state: overlay tint using `--color-surface-container` at reduced opacity; lock icon (text glyph or SVG, no icon library); title rendered in `--color-on-surface-variant`
- Unlocked state: full contrast; guillemet `«` prefix on the active/in-progress lesson in `--color-primary`
- No shadows. Tonal distinction only.

### SubComponentList (`components/lessons/SubComponentList.tsx`)
- `'use client'` — wraps `useOptimistic` state
- Renders an ordered vertical list of SubComponentItem
- List gap: 8px (sm) between items

### SubComponentItem (`components/lessons/SubComponentItem.tsx`)
- Layout: horizontal flex, items aligned center, gap 12px
- Left: completion toggle (button, 44px × 44px touch target)
  - Incomplete: circle outline in `--color-outline`, `--color-on-surface` icon (checkmark placeholder or dash)
  - Complete (optimistic or confirmed): filled circle in `--color-primary`, white checkmark; `aria-pressed="true"`; `disabled` attribute set
- Center: sub-component title (body-md) + kind chip (label-sm)
- Kind chip colors (tonal, warm only):
  - `explainer`: `--color-surface-container-high` background, `--color-on-surface-variant` text, label "reading"
  - `practice`: `--color-surface-container-highest` background, `--color-on-surface-variant` text, label "practice"
  - `writing`: `--color-surface-container-highest` background, `--color-on-surface-variant` text, label "writing"
- Completed item: title in `--color-on-surface-variant` (muted), not struck through (strikethrough is high-energy/gamified — avoid)
- Border radius on chip: `--radius-full` (pill)

### LockBadge (`components/ui/LockBadge.tsx`)
- Pill badge: `--color-surface-container-high` background, `--color-on-surface-variant` text
- Label: "locked" (sentence case)
- Font: label-sm (Work Sans 13px)
- Border radius: `--radius-full`
- Used on locked level cards and locked level page header

### ProgressBar (inline in lesson page)
- Thin horizontal bar, 4px height
- Track: `--color-surface-container-high`
- Fill: `--color-primary`
- Border radius: `--radius-full`
- Position: below lesson title, above sub-component list
- Label (label-sm, muted): "{N} of {total} complete" — positioned right-aligned above or below bar
- No circular rings (per DESIGN.md: "avoid circular rings which can feel high-pressure")

## Layout Contracts

### Level Page (`/levels/[levelSlug]`)
- Outer container: `max-width: 1040px`, horizontally centered, side padding 24px desktop / 20px mobile
- Page header: level name in display-lg (Literata 48px), level description in body-md
- Lesson card grid: single column on mobile; 2-column on tablet+; card gap 16px vertical
- Locked level page: full page shows lock badge + "unlock after placement diagnostic" body copy; lesson cards rendered but visually locked (no interactive state on locked cards)

### Lesson Page (`/levels/[levelSlug]/lessons/[lessonId]`)
- Content column: `max-width: 720px`, centered, side padding 24px desktop / 20px mobile
- Lesson header: title in headline-lg, estimated time in label-sm (muted), progress bar below
- Back link: "Back to French 1" — body-md weight 400, `--color-primary` color, above the title
- Sub-component list: full width within 720px content column

## Interaction Contracts

### Mark complete toggle
- Trigger: click or keyboard Enter/Space on SubComponentItem toggle button
- Immediate effect: optimistic UI — button switches to filled/completed state before server round-trip (`useOptimistic`)
- Pending state: no spinner; button stays in completed state (optimistic)
- Server confirmed: `revalidatePath` fires; Server Component re-renders with ground-truth; optimistic state reconciles silently
- Error state (server action throws): revert optimistic state; show inline error message below the item: "Couldn't save progress. Try again." in `--color-error`, label-sm
- Double-click safety: button is `disabled` once optimistic state is complete; idempotent upsert handles any race

### Locked level card
- Not clickable (no `href` or `onClick`)
- Cursor: `default` (not pointer)
- Keyboard: not focusable (`tabIndex="-1"` or `aria-disabled="true"`)
- Tooltip or inline label: "Locked — complete French 1 to unlock" (shown on hover via `title` attribute for Phase 3; enhanced tooltip in Phase 9)

### Skeleton loaders
- Shown while Server Component data fetches (streaming or Suspense boundary)
- Warm-tinted skeleton: base color `--color-surface-container`, shimmer color `--color-surface-container-high`
- Skeleton shapes mirror actual card dimensions; no cold grey

| Mark complete button (done) | "Done" |

## Accessibility Contract

- All interactive elements keyboard-accessible (Tab, Enter, Space)
- `aria-pressed` on sub-component toggle buttons (binary state)
- `aria-label` on icon-only elements (lock icon, progress bar)
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="{total}"`
- Locked cards: `aria-disabled="true"` with descriptive `aria-label` including locked reason
- Color is never the sole differentiator — completed state uses both color change AND text change ("Mark complete" → "Done")
- WCAG AA contrast: primary button `#a03e40` on white passes (ratio 5.4:1, verified per REQUIREMENTS.md SEC-06)
- Skeleton loaders: `aria-busy="true"` on the loading region; `aria-label="Loading lesson…"`

## Dark Mode Contract

All colors switch via `.dark` class on `<html>` (next-themes, already wired in Phase 1).
No Phase 3 components introduce hardcoded hex values — CSS custom properties handle both modes automatically.

Key dark mode notes:
- Lesson cards: `--color-surface-container-low` dark value `#221e1c`
- Primary action: `--color-primary` dark value `#ffb866` (warm orange) — matches DESIGN.md "candle-lit" dark palette
- Progress bar fill: `#ffb866` in dark mode (still warm, high contrast on dark surface)
- Skeleton shimmer: `--color-surface-container` dark `#262220` / `--color-surface-container-high` dark `#312c29`

## What This Phase Does NOT Introduce

- Green (`--color-tertiary`): reserved for Phase 5 correct-answer feedback only
- Circular progress rings: DESIGN.md explicitly prohibits (high-pressure feel)
- Heavy drop shadows: tonal layers only; shadow only on dropdowns/mobile nav
- Sequential lesson gating within French 1: all French 1 lessons are freely jumpable (D-L04)
- Score display: binary done/not-done only (D-L02); no percentage scores, no stars
- Any new fonts or icon libraries
