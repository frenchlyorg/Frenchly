Create src/components/practice/WrittenCard.tsx as a 'use client' component.

Props interface WrittenCardProps:
  - problem: WrittenProblem
  - subComponentId: string
  - isCompleted: boolean
  - initialFeedback?: string | null  (loaded from DB on revisit — D-09)
  - onComplete: (id: string) => void

State:
  - text: string — textarea content (initial '')
  - feedback: string | null — initialized from initialFeedback ?? null (D-09)
  - loading: boolean — true while fetch is in flight
  - done: boolean — initialized from isCompleted (D-08)

Word count (D-02): derive from text state — `text.trim() === '' ? 0 : text.trim().split(/\s+/).length` — no state, computed inline. Display below textarea: "N words" using font-label text-[13px] text-on-surface-variant. No cap shown.

Auto-resize textarea (D-01): use a ref on the textarea element. In onChange handler, after setText(e.target.value), set `ref.current.style.height = 'auto'` then `ref.current.style.height = ref.current.scrollHeight + 'px'`. No fixed rows prop. Add `resize-none` class to prevent manual resize handle.

Submit button (D-03): text "Check my writing". Disabled when `text.trim() === '' || loading || done`. When loading, render a spinner SVG inside the button (simple 16px spinning circle using animate-spin Tailwind class) and also keep the text. aria-busy={loading}. Use the primary button token: `bg-primary text-white font-label text-[13px] rounded-[8px] disabled:opacity-40 disabled:cursor-not-allowed`.

Textarea disabled state (D-04 during loading, D-08 when done): disabled={loading || done}. Apply muted appearance when done: `disabled:opacity-70 disabled:cursor-not-allowed bg-surface-container-low`.

handleSubmit (useCallback — D-05, D-06, D-07 execution order):
  1. Guard: if `!text.trim() || loading || done` return early
  2. setLoading(true)
  3. try:
     - const res = await fetch('/api/check-writing', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ subComponentId, text }) })
     - const data = await res.json()
     - if data.rateLimited: setFeedback("You've used all your writing checks for today — come back tomorrow!")
     - else: setFeedback(data.feedback ?? "We couldn't check that right now — keep going!")
  4. catch: setFeedback("We couldn't check that right now — keep going!")
  5. finally: setLoading(false); setDone(true); onComplete(subComponentId); await markSubComponentComplete(subComponentId)
  
Note: onComplete and markSubComponentComplete both called in finally — this fires AFTER setFeedback (D-05). Both rate-limit and error paths also land here via try/catch/finally (D-06, D-07).

Render structure:
  - Card container: `bg-surface-container-low border border-outline-variant rounded-[16px] p-6 mt-4`
  - Prompt: `font-body text-[18px] leading-8 text-on-surface mb-4` — problem.prompt text
  - Textarea: full-width, auto-resize (ref), tokens from FillInInput pattern, disabled when loading||done
  - Word count: below textarea, `font-label text-[13px] text-on-surface-variant mt-1`
  - Submit button: shown only when !done (hide after completion, D-08)
  - Feedback display (when feedback !== null): warm neutral box — `mt-4 rounded-lg bg-surface-container border border-outline-variant px-4 py-3` with `font-body text-[16px] text-on-surface` text. NEVER use text-tertiary, bg-tertiary, border-tertiary, or any green token here (CLAUDE.md rule 3, UX-10).
