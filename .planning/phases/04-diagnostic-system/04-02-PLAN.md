<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated student → profiles table | Students may UPDATE their own profile (username only). The watermark/placement columns must NOT be writable here. |
| authenticated student → diagnostic_attempts / diagnostic_answers | Students read/write only their own rows; cross-student access must be blocked by RLS. |
| client query → diagnostic_questions.correct_answer | Answer key sits in a content table readable by authenticated; must not leak via client projections. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-EoP-01 | Elevation of Privilege | profiles.unlocked_through_level_number | mitigate | Column added but excluded from the authenticated UPDATE grant; only service_role (admin client) may write it (carries Phase 3 T-03-01 guard). Enforced at the grant level in this migration. |
| T-04-ID-01 | Information Disclosure | diagnostic_questions.correct_answer | mitigate | RLS allows authenticated read of the table; answer key protected by app-level column-projection discipline (Plans 03/05 select only id, question_text, type, options, lesson_tag). SQL comment documents this constraint. |
| T-04-Tamp-01 | Tampering | concurrent in_progress attempts | mitigate | Partial unique index `where status='in_progress'` makes a second concurrent attempt insert fail atomically (Pitfall 3). |
| T-04-Tamp-02 | Tampering | cross-student attempt/answer read or write | mitigate | RLS policies scope all attempt rows to `(select auth.uid()) = user_id`; answer rows scoped via exists() join to parent attempt ownership. |
| T-04-SC | Tampering | npm/db installs | accept | No package installs; only a DB migration push. 04-RESEARCH.md Package Legitimacy Audit: no new packages. |
