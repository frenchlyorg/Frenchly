## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, chip internal padding |
| sm | 8px | Inline label gaps, compact stack between answer options |
| md | 16px | Default element spacing, gap between question and options |
| lg | 24px | Card internal padding, section stack within diagnostic step |
| xl | 32px | Gap between major blocks (question card → progress bar) |
| 2xl | 48px | Vertical gap between result screen sections |
| 3xl | 64px | Page-level top/bottom padding on diagnostic flow |

Exceptions:
- Touch targets for MC option buttons: minimum 44px height (WCAG 2.5.5, matches existing form inputs)
- Cooldown countdown block: 24px internal padding matches card padding (lg)
- Soft-timer badge: 8px horizontal padding, 4px vertical (xs/sm rhythm)

Content max-width: 720px (diagnostic flow is lesson-width, not dashboard-width — matches `--spacing-content-max`)

## Color

All hex values from DESIGN.md / globals.css tokens. No ad-hoc values.

| Role | Token | Hex (light) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `background` / `surface` | `#fff8f5` | Page background, diagnostic step background |
| Secondary (30%) | `surface-container-low` / `surface-container` | `#fbf2ed` / `#f5ece7` | Question card background, option button surfaces |
| Accent (10%) | `primary` | `#a03e40` | Primary CTA buttons, active progress fill, selected-option border |
| Correct feedback | `tertiary` | `#006c4a` | Correct-answer state on options only — nowhere else |
| Incorrect / error | `error` | `#ba1a1a` | Wrong-answer state on options, error messages |
| Destructive | `error` | `#ba1a1a` | No destructive actions in this phase (see Copywriting) |

Accent (`primary`) reserved for:
1. "Start diagnostic" / "Submit answer" / "Next question" / "Retry" primary buttons
2. Active/selected answer option border highlight (2px border-primary before submission)
3. Progress bar fill (thin horizontal, showing questions completed / total)
4. Timer badge outline (soft border only, not fill)

Green (`tertiary`) reserved for:
- Correct-answer feedback state on an option button (bg-tertiary/10 tint + border-tertiary + checkmark icon)
- Soft accent note text is `text-secondary` (warm amber), NOT green

Cool tones: none. Warm palette strictly maintained.

## Component Inventory

New components to build for this phase. All use existing design tokens.

### DiagnosticGate (placement gate)
- Full-page interstitial shown to first-time users before any lesson/dashboard access
- Surface: `bg-surface` page + centered card `bg-surface-container-low` max-w-[480px]
- Guillemet decoration: `« »` flanking the heading in `font-heading text-primary`
- Copy: "Before you begin" heading + single paragraph explaining the placement diagnostic
- Single primary CTA: "Start placement diagnostic"
- No dismiss/skip affordance (gate is mandatory per D-P01/D-P02)

### DiagnosticProgress (progress bar + timer)
- Thin horizontal bar, 4px height, `bg-primary` fill on `bg-surface-container-high` track
- Sits above the question card, full content width (720px)
- Right-aligned soft-timer badge: `font-label text-[13px] text-on-surface-variant` — shows elapsed time "2:14"
- Left-aligned question counter: `font-label text-[13px] text-on-surface-variant` — "Question 3 of 10"

### DiagnosticQuestionCard
- Container: `bg-surface-container-low border border-outline-variant rounded-[16px] p-6`
- Question stem: `font-body text-[18px] text-on-surface leading-8`
- Primary visual anchor: the question stem — largest body text (18px) on the surface, full-width, top of card; answer options sit below in the type scale.
- No shadow (tonal depth only, per DESIGN.md elevation rules)

### MCOptionButton (multiple choice answer)
States and token mapping:

| State | Background | Border | Text |
|-------|-----------|--------|------|
| Default | `bg-surface-container` | `border-outline-variant` | `text-on-surface` |
| Hover (pre-submit) | `bg-surface-container-high` | `border-outline` | `text-on-surface` |
| Selected (pre-submit) | `bg-surface-container-low` | `border-primary` (2px) | `text-on-surface` |
| Correct (post-submit) | `bg-tertiary/10` | `border-tertiary` (2px) | `text-on-surface` |
| Incorrect (post-submit) | `bg-error/10` | `border-error` (2px) | `text-on-surface` |
| Disabled (other options after submit) | `bg-surface-container` | `border-outline-variant` | `text-on-surface-variant opacity-60` |

- Height: min 44px, padding px-4 py-3, rounded-[8px] (radius DEFAULT)
- Icon: Lucide `Check` (16px, text-tertiary) on correct; Lucide `X` (16px, text-error) on incorrect
- Full-width stacked list with `gap-4` between options

### FillInInput (fill-in-the-blank)
- Inherits existing input pattern from LoginForm: `border border-outline rounded px-3 py-2 bg-surface-container-low text-on-surface font-body text-sm min-h-[44px] focus:outline-none focus:border-b-[3px] focus:border-primary transition-all`
- Post-submit correct state: `border-tertiary border-b-[3px]`
- Post-submit incorrect state: `border-error border-b-[3px]`
- Soft accent note (accent-insensitive match): inline `p` below input, `text-secondary font-label text-[13px]` — "Correct — watch the accents: café, not cafe"

### DiagnosticResult (placement result screen)
- Layout: centered, max-w-[480px], `bg-surface-container-low rounded-[16px] p-8`
- Heading: Guillemet-framed level name — `« French 1 »` or `« French 2 »` in `font-heading text-[28px] text-on-surface`
- Subheading: encouraging message (no raw percentage) — `font-body text-[16px] text-on-surface-variant`
- CTA: "Go to my lessons" — primary button

### DiagnosticResultFail (end-of-level fail screen)
- Score line: `font-label text-[13px] text-on-surface-variant` — "You scored X of 10" (percentage not shown, only correct count)
- Weak-area section: labeled `« Review these topics »` header in `font-heading text-[18px] font-semibold text-on-surface`; bulleted list of linked lesson titles using `font-body text-[16px] text-primary hover:underline`
- Cooldown block: `bg-surface-container border border-outline-variant rounded-[8px] p-6`
  - Copy: "Retry available in" + live countdown `font-heading text-[28px] text-on-surface`
  - Retry button: disabled state with `opacity-40 cursor-not-allowed` while cooldown active; enabled state is standard primary button

### CooldownCountdown (live timer)
- Client component (`'use client'`) — updates every minute via `setInterval`
- Display format: "2h 14m" — no seconds shown (reduces anxiety)
- When countdown reaches 0:00: button re-enables, copy changes to "Retry diagnostic"

## Interaction Contracts

### Placement gate flow
1. First-time user hits `/dashboard` or `/levels/*` → server-component guard detects no completed placement → renders `DiagnosticGate` interstitial
2. User clicks "Start placement diagnostic" → navigates to `/diagnostic/placement`
3. 10 questions, one at a time (no scrolling through all)
4. After each answer submission, brief feedback state (correct/incorrect) shown for 1.2s before auto-advancing — or a "Next question" button appears immediately for fill-in (user controls pace)
5. Final question submitted → server action scores → result screen
6. "Go to my lessons" → dashboard (gate no longer shown)

### Question navigation
- One question visible at a time (no question list sidebar)
- No back navigation — answers are final once submitted (keeps grading simple, prevents second-guessing)
- Progress bar advances on each submission

### Answer submission
- MC: clicking an option immediately marks it selected; a "Submit answer" button confirms (prevents accidental taps)
- Fill-in: text input + "Submit answer" button; enter key also submits
- After submission: all options disabled, correct/incorrect states revealed, "Next question" button appears

### Resumable state
- Returning to an in-progress diagnostic: same question set resumes, previously answered questions are not re-shown
- Progress bar reflects already-answered count
- Timer resets to 0:00 on resume (elapsed time is informational only)

### End-of-level diagnostic availability
- "Take end-of-level diagnostic" CTA appears on the level page only after all sub-components complete
- Before all sub-components complete: CTA absent (not locked badge — just not shown yet)

### Cooldown state
- Retry button disabled with countdown displayed
- No other UI blocked by the cooldown (other levels, lessons unaffected)
- When cooldown expires client-side: button enables without page reload

## Accessibility Contract

- All interactive elements: minimum 44x44px touch target (matches existing input pattern)
- MC options: `role="radio"` within a `role="radiogroup"` — or styled `<button>` elements with `aria-pressed` state
- Correct/incorrect icons: `aria-hidden="true"` on Lucide icons; result conveyed via text ("Correct" / "Not quite")
- Progress bar: `role="progressbar" aria-valuenow={current} aria-valuemax={total} aria-label="Diagnostic progress"`
- Cooldown countdown: `aria-live="polite"` region so screen readers announce when timer reaches 0
- Soft accent note: rendered as `<p>` after the input, associated via `aria-describedby`
- Focus management: after auto-advance, focus moves to the new question card heading
- Error messages: `role="alert"` (matches existing pattern from LoginForm)
- Keyboard navigation: Tab through options, Space/Enter to select, Enter to submit
