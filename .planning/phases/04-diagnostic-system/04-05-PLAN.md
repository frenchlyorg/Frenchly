## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → startEndOfLevelDiagnostic / submitDiagnosticAnswer | Untrusted start/answer requests cross here; user + cooldown + grading enforced server-side. |
| Server Action → profiles (watermark advance) | Privilege boundary — only service_role may advance the watermark on pass. |
| client → retry button | A client cannot bypass the cooldown; the server re-checks isCooldownActive before creating a retry attempt. |
| Server Component → diagnostic_questions | Answer key must not cross into the RSC payload. |
