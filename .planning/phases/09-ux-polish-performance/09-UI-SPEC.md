## Component Inventory

### PostLessonBar (new — extracted into SubComponentList or separate file)

| Prop | Type | Notes |
|------|------|-------|
| — | — | No props; triggered internally when `allDone` becomes true |

States:
- **Filling:** bar animates 0% → 100% over 1.5 seconds; message visible below bar
- **Complete:** `router.push(/levels/[levelSlug])` fires after 1.5s

Variants: none (single appearance)

Implementation notes:
- CSS transition: `transition-all duration-[1500ms]` on the fill div, triggered by `useEffect` setting width to 100% after mount
- `useRouter` from `next/navigation` for navigation
- `levelSlug` must be accessible in SubComponentList (passed as prop from lesson page)
- Message selected via `useMemo(() => messages[Math.floor(Math.random() * messages.length)], [])` — stable across re-renders

### AccordionShell (pattern applied to SubComponentItem)

SubComponentItem gains accordion behavior. Not a separate component — accordion state lives in SubComponentList.

SubComponentList props change (internal state):
```
openId: string | null   // which sub-component is expanded
setOpenId: setter
```

SubComponentItem gains:
```
isOpen: boolean         // whether this item's content is visible
onToggle: () => void    // called when header is clicked
```

Accordion header (new `<button>` wrapper):
- `aria-expanded={isOpen}`
- `aria-controls={`content-${id}`}`
- `id={`header-${id}`}`
- Contains: completion circle + title + kind chip + action label
- Full-width clickable: `flex items-center gap-3 py-3 w-full text-left`

Accordion content region:
- `id={`content-${id}`}`
- `role="region"`
- `aria-labelledby={`header-${id}`}`
- Visibility: `hidden` when collapsed, visible when `isOpen`
- Animation: CSS `max-height` transition (0 → auto via max-height trick) — chosen by Claude over display toggle because `display:none` breaks smooth animation and the max-height approach avoids layout shift. Use `overflow-hidden transition-[max-height] duration-200 ease-out` with `max-h-0` (closed) / `max-h-[2000px]` (open). 200ms is fast enough to feel responsive without being jarring.

Completion checkmark on collapsed header:
- Completion circle remains in header row at all times (not hidden when collapsed)
- `bg-primary border-primary` styling visible in collapsed state

States per accordion item:
- **Closed, incomplete:** header visible, content hidden, circle = outline dash
- **Open, incomplete:** header visible, content visible, circle = outline dash
- **Closed, complete:** header visible, content hidden, circle = filled coral checkmark
- **Open, complete (manual re-open):** header visible, content visible, circle = filled coral checkmark

## Interaction Contracts

### D-AC — Accordion Behavior

**Initial state on lesson load:**
- `openId` = `subComponents[0].id` (first item open, rest closed)
- If `subComponents[0]` is already complete, first item opens anyway (user can still re-read it)

**User clicks accordion header:**
- If clicking the currently open item: `setOpenId(null)` — collapses all
- If clicking a closed item: `setOpenId(id)` — opens that item, previous item collapses (one-open-at-a-time model)
- Completed items: clickable, re-opens to show content (manual review)
- The accordion header button must be the entire header row (full-width), not just the chevron/icon

**Auto-advance on complete:**
1. `onComplete(id)` fires (from button click, practice answer, or writing submission)
2. `setOptimisticCompleted(id)` runs (existing)
3. In the same handler: derive `nextId` = `subComponents` sorted by `position`, find the entry after `id` that is NOT yet complete
4. `setOpenId(nextId ?? null)` — if no next incomplete item, close all (post-lesson bar will appear)
5. When `allDone` becomes true (after optimistic update): `PostLessonBar` renders and begins fill

**Collapse/expand animation:**
- Property: `max-height`
- Closed: `max-h-0 overflow-hidden`
- Open: `max-h-[2000px] overflow-hidden` (generous ceiling covers any content height)
- Transition: `transition-[max-height] duration-200 ease-out`
- `prefers-reduced-motion`: wrap in `motion-reduce:transition-none` — animation disabled, snap open/close

**Keyboard behavior:**
- `Tab` moves between accordion header buttons
- `Enter` / `Space` on focused header: toggle open/close
- No arrow-key navigation required (WCAG 2.1 APG accordion pattern allows Tab-only for simple accordions)

### D-PL — Post-lesson Loading Bar

**Trigger:** `allDone === true` (derived from optimistic state in SubComponentList)

**Rendering sequence:**
1. `allDone` becomes true → `PostLessonBar` mounts (`useEffect` triggers on mount)
2. Frame 1: bar renders at `width: 0%`
3. Frame 2 (next tick via `requestAnimationFrame` or `setTimeout(0)`): set `width: 100%` — CSS transition takes over
4. After 1500ms: `router.push('/levels/${levelSlug}')` fires

**Bar anatomy:**
```
[fixed top-0 left-0 right-0 z-50]
  [track: h-1 w-full bg-surface-container-high]
    [fill: h-full bg-primary transition-all duration-[1500ms] ease-in-out]
  [message: mt-2 text-center font-label text-[13px] text-on-surface-variant]
```

The message div sits in normal flow BELOW the fixed bar — it is `position: static` inside a `fixed` container. The fixed container height = `auto` to accommodate both bar and message. This does push page content down by the message height (~32px) which is acceptable for a 1.5s transition — no layout shift workaround needed.

**Message selection:** one of 5 strings chosen at random on component mount (`useMemo` with empty dep array). Does not change during the fill animation.

**Navigation:** `useRouter` from `next/navigation`. `router.push` (not `router.replace`) so back-button returns to lesson.

**Replaces:** the existing `{allDone && <div className="mt-8 p-6 ...">Lesson complete</div>}` block in SubComponentList — remove it entirely.

**Reduced motion:** bar still displays and navigates; only the CSS fill animation is removed (`motion-reduce:transition-none` on fill div). Navigation still fires after 1500ms.

### D-SK — Skeleton Loaders

**Trigger:** Next.js App Router — `loading.tsx` co-located with `page.tsx` at each route segment. The router automatically renders the skeleton while the Server Component suspends.

**Files to create:**
- `src/app/levels/[levelSlug]/loading.tsx`
- `src/app/levels/[levelSlug]/lessons/[lessonId]/loading.tsx`
- `src/app/dashboard/loading.tsx`
- `src/app/diagnostic/placement/loading.tsx`
- `src/app/diagnostic/end-of-level/[levelSlug]/loading.tsx`

**Animation:** `animate-pulse motion-reduce:animate-none` on every skeleton block. Pulse = Tailwind's opacity 1→0.5→1 cycle (no shimmer gradient).

**Timing:** skeleton displays until Server Component resolves. No minimum display time — show as short as needed.

**Fidelity contract:**
- Each skeleton must mirror the real page's structural layout (same grid, same number of blocks, same proportions)
- Skeleton blocks do NOT contain text or icons
- Container max-widths must match the real page (`max-w-[1040px]` for level/dashboard, `max-w-[720px]` for lesson/diagnostic)

### D-GM — Active-Lesson Marker

**Computation (server-side, in LevelPage):**

```
// Already available: lessons (position-sorted), completedSet
const activeLessonId = (() => {
  for (const lesson of lessons) {  // lessons already ordered by position ASC
    const lessonSubIds = (lesson.sub_components ?? []).map(s => s.id)
    const hasIncomplete = lessonSubIds.some(id => !completedSet.has(id))
    if (hasIncomplete) return lesson.id
  }
  return null  // all complete — no active marker
})()
```

Then pass `isActive={lesson.id === activeLessonId}` to each `<LevelCard>`.

**Edge cases:**
- Level locked (`isLocked === true`): `isActive` may be true but guillemet is suppressed by existing `{isActive && !isLocked}` guard in LevelCard — no change needed
- Level complete (all sub-components done): `activeLessonId = null`, no card gets `isActive={true}`
- First visit (zero progress): first lesson gets `isActive={true}`
- Lesson with zero sub-components: `hasIncomplete` = false (empty `some` = false) — lesson is skipped, never active

**No client-side re-computation.** This is pure server data derivation — the value is stable for the page render.

### D-A11Y — Accessibility Contracts

**Accordion button:**
```html
<button
  type="button"
  aria-expanded={isOpen}
  aria-controls={`content-${id}`}
  id={`header-${id}`}
  className="flex items-center gap-3 py-3 w-full text-left ..."
>
  {/* circle + title + kind chip + action label */}
</button>
<div
  id={`content-${id}`}
  role="region"
  aria-labelledby={`header-${id}`}
  className={isOpen ? '' : 'hidden'}
>
  {/* accordion content */}
</div>
```

**Post-lesson bar:**
- Bar: `role="progressbar" aria-valuenow={progress} aria-valuemin={0} aria-valuemax={100} aria-label="Loading next lesson"`
- Message: visible text — no aria-live needed (user focus is on the visual transition)

**Skeleton loaders:**
- Wrap in `<div aria-hidden="true">` — screen readers should not announce skeleton blocks
- No `role="status"` or `aria-busy` required (Next.js router handles loading state announcement)

**Lighthouse targets:**
- Performance: ≥ 85 mobile (throttled)
- Accessibility: no new violations introduced
- Manual check required: `bg-primary` (#a03e40) on `bg-surface-container-lowest` (#ffffff) — confirm passes WCAG AA (4.5:1 for normal text). Existing usage confirmed passing in prior phases.
