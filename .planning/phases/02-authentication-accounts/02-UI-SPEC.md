| Token | Value | Usage |

## Component Specifications

### Input Fields

All inputs follow the "lightly boxed" pattern (CLAUDE.md rule 5, DESIGN.md components):

- **Resting:** 1px border `--color-outline-variant`, background `--color-surface-container-low`, radius `--radius` (8px)
- **Focus:** bottom border widens to 3px solid `--color-primary`; top/left/right borders remain 1px `--color-outline-variant`; no glow/box-shadow
- **Error state:** all 4 borders become 1px solid `--color-error`; error text appears immediately below the field in Be Vietnam Pro 14px `--color-error`
- **Disabled:** background `--color-surface-container`, border `--color-outline-variant`, text `--color-on-surface-variant`, cursor not-allowed
- **Placeholder text:** `--color-on-surface-variant` at 70% opacity
- **Height:** 44px minimum (WCAG touch target)
- **Padding:** 12px horizontal, 10px vertical

Password inputs include a show/hide toggle button (eye icon or "show"/"hide" text link in Work Sans 13px) aligned to the right edge of the input, using `--color-on-surface-variant`. Revealing password briefly to verify before submit is standard accessibility.

### Primary Button

- **Fill:** `--color-primary` (#a03e40 light / #ffb866 dark)
- **Text:** `--color-on-primary` (#ffffff light / #3a2400 dark), Work Sans 14px weight 500
- **Height:** 44px, full width of form container (480px max)
- **Radius:** `--radius` (8px)
- **Hover:** background darkens 10% (light: ~#8a3436; dark: ~#e6a55a) — no shadow
- **Active:** 5% additional darken
- **Disabled:** 40% opacity, cursor not-allowed
- **No heavy box shadow**

### Secondary / Ghost Button

Used for "Forgot password?", "Already have an account? Log in", "Cancel" in account deletion:
- Text-only: `--color-primary` colored text, no fill, no border
- Hover: `--color-primary` at 10% opacity background (warm tint)
- Size: Work Sans 14px weight 400

### Destructive Button (Delete account)

- Border: 1px solid `--color-error`
- Text: `--color-error`
- Fill: transparent (ghost)
- Hover: `--color-error` at 8% opacity background
- Active/enabled only when "delete" typed in confirmation input — otherwise disabled (40% opacity, cursor not-allowed)

### Form Card

- Background: `--color-surface-container-low`
- Border: none (tonal depth, no border needed against background)
- Radius: `--radius-lg` (16px)
- Padding: 32px vertical, 24px horizontal (desktop); 24px all sides (mobile)
- Max-width: 480px, centered horizontally
- No box shadow. Source: DESIGN.md elevation.

## Screen Specifications

### /signup — Sign-up page

**Layout:** Single centered column. Form card max-width 480px, vertically centered with top offset ~80px from nav.

**Elements (top to bottom):**
1. Page heading: "Create your account" — Literata 28px/600
2. Sub-copy: "Join Frenchly and start learning French today." — Be Vietnam Pro 16px, `--color-on-surface-variant`
3. Form card:
   - Field: Email (label "Email address")
   - Field: Username (label "Username", hint: "3–20 characters: letters, numbers, underscores")
   - Field: Password (label "Password", hint: "At least 12 characters including one number", show/hide toggle)
   - Primary button: "Create account"
4. Below card: "Already have an account? [Log in]" — Be Vietnam Pro 14px + inline ghost link

**Error messages (inline under field):**
- Email taken: "Hmm, that email's taken — already a Frenchly member?"
- Username taken: "That username is gone — try adding your initial or a number."
- Username format violation: "Username must be 3–20 characters: letters, numbers, underscores only."
- Username profanity: "That username isn't available. Try something else."
- Password too short/no number: "Make it at least 12 characters with a number — your account will thank you."
- Generic fallback: "Something went wrong. Please try again."

Source: CONTEXT.md D-11, D-12.

### /login — Login page

**Layout:** Single centered column. Same card geometry as sign-up.

**Elements (top to bottom):**
1. Page heading: "Welcome back" — Literata 28px/600
2. Form card:
   - Field: Email address
   - Field: Password (show/hide toggle)
   - "Forgot password?" — ghost button, right-aligned under password field
   - Primary button: "Log in"
3. Below card: "New to Frenchly? [Create an account]" — Be Vietnam Pro 14px + inline ghost link

**Error messages:**
- Auth failure: "Email or password incorrect." (vague — CONTEXT.md D-13)
- Rate limit lockout: "Too many attempts. Please wait 15 minutes before trying again." (shown as a non-field message at top of form card, using `--color-on-surface-variant` text, NOT error red — this is a security gate, not a user mistake)

Source: CONTEXT.md D-11, D-12, D-13, D-19.

**Lockout message styling:** Amber/warm info treatment (Be Vietnam Pro 14px, `--color-secondary-container` background tint at 20%, `--color-on-surface` text) — distinct from red error styling. Placed above the email field inside the form card.

### /account — Account settings page

**Layout:** Page container 1040px. Single column, left-aligned sections with 32px stack gaps.

**Sections:**
1. Page heading: "Account settings" — Literata 28px/600
2. **Password section** (card): Change password
   - Field: Current password
   - Field: New password (hint: 12+ chars, one number)
   - Field: Confirm new password
   - Primary button: "Update password"
3. **Danger zone section** (card, subtle error-tinted border: 1px solid `--color-error` at 30% opacity):
   - Section label: "Delete account" — Work Sans 13px/500 `--color-error`
   - Description: "This will permanently remove your account. Your learning progress cannot be recovered." — Be Vietnam Pro 14px `--color-on-surface-variant`
   - Confirmation input (label: "Type delete to confirm"):
     - Placeholder: "delete"
     - Standard input styling (not red) until confirmed
   - Destructive button: "Delete my account" — disabled until field value === "delete" (case-sensitive)

Source: CONTEXT.md D-14, D-15.

### /dashboard — Dashboard stub

**Layout:** 1040px container. Top section only (stub state).

**Elements:**
1. Welcome message: "Welcome back, [username]." — Literata 24px/500
2. Placeholder card: "Your lessons are coming soon." with sub-copy "Level 1 content launches in a future update." — Be Vietnam Pro 16px, `--color-on-surface-variant`, inside a surface-container card with dashed `--color-outline-variant` border.
3. No green colors, no confetti, no celebration — warm neutral stub. Source: DESIGN.md (green = correct-answer only).

### /admin — Admin stub (protected)

**Layout:** 1040px container.

**Elements:**
1. Page heading: "Admin area" — Literata 28px/600
2. Description card: "Content management is coming soon. This area will allow lesson editing and student management." — Be Vietnam Pro 16px `--color-on-surface-variant`

## Accessibility Contract

- All form inputs have explicit `<label>` elements with `htmlFor` association (no placeholder-as-label)
- Error messages are wired to inputs via `aria-describedby` pointing to the error element's id
- Error elements use `role="alert"` or `aria-live="polite"` so screen readers announce inline errors
- Primary button gets `aria-disabled` + `disabled` when in disabled state
- Password show/hide toggle: `aria-label="Show password"` / `aria-label="Hide password"` toggled programmatically
- Lockout message: `role="alert"` so it announces immediately when it appears
- Delete confirmation input: `aria-label="Type delete to confirm account deletion"`
- Tab order: logical top-to-bottom, matching visual order
- Focus ring: 2px offset ring in `--color-primary` (Tailwind `focus-visible:ring-2 focus-visible:ring-[--color-primary]`)

Source: REQUIREMENTS.md SEC-06, SEC-07.
