<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser → React hydration | next-themes injects class on <html> before React hydrates; suppressHydrationWarning required |
| localStorage → theme state | User-controlled storage; value is "light" or "dark" only — no injection risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | ThemeToggle (client) | accept | No auth in Phase 1; theme toggle is purely cosmetic |
| T-02-02 | Tampering | localStorage theme value | accept | next-themes reads and validates "light"/"dark" strings; invalid values fall back to defaultTheme="light"; no security impact |
| T-02-03 | Information Disclosure | nav.tsx Link hrefs | accept | All hrefs point to public routes only; no authenticated paths in Phase 1 |
| T-02-04 | XSS | Nav link text / logo text | mitigate | All text is hardcoded string literals, not user input; React escapes JSX text by default |
| T-02-SC | Tampering | npm installs | mitigate | Only next-themes and lucide-react added — both APPROVED in Plan 01 Package Legitimacy Audit |
</threat_model>
