<task type="checkpoint:human-action">
  <name>Step 1: Mitigate Supabase free-tier auto-pause [DASHBOARD: cron-job.org]</name>
  <what-built>Nothing automated — configure a free uptime pinger to prevent Supabase from auto-pausing during low-traffic periods.</what-built>
  <how-to-verify>
    Supabase free tier auto-pauses projects after 7 days of zero activity. A paused project takes ~30 seconds to wake up on the next request — visible as a loading delay for the first user.

    Mitigation: Set up a free pinger that hits frenchly.org every 5 days, keeping the project awake.

    1. Go to https://cron-job.org and create a free account.
    2. Create a new cron job:
       - URL: https://frenchly.org (or your Vercel preview URL until frenchly.org is live — update after Step 10)
       - Schedule: Every 5 days (or use the expression: 0 9 */5 * *)
       - Request type: GET
       - Enable the job.
    3. Confirm the job is active. It will make a GET request every 5 days, keeping Supabase alive.

    Alternative (no account needed): Use https://uptimerobot.com — free tier monitors every 5 minutes, which also prevents auto-pause.

    When to upgrade Supabase to Pro ($25/month):
    - Auto-pause fires despite the pinger (unlikely), OR
    - You have ~50+ regular weekly users and want guaranteed uptime without the pinger dependency.
    Upgrade path: Supabase Dashboard -> Settings -> Billing -> Upgrade to Pro.

    Expected result: Cron job created and active. Supabase project will not auto-pause as long as the pinger runs.
  </how-to-verify>
  <resume-signal>Type "auto-pause mitigation set" when the pinger is configured.</resume-signal>
</task>

    1. Log in to https://www.ionos.com -> Domains & SSL -> frenchly.org -> DNS.

       - Type: A  |  Host: @  |  Points to: 76.76.21.21  |  TTL: 3600

    3. Add Resend records (from Step 3, exact values from your Resend dashboard):
       - Type: TXT  |  Host: resend._domainkey  |  Value: [Resend DKIM value]  |  TTL: 3600

         If an SPF TXT record for @ already exists, append the Resend include to it rather than creating a duplicate.
    4. Save all records.

    You can check propagation at: https://www.whatsmydns.net/#A/frenchly.org

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Vercel/Next.js | Untrusted user input enters via sign-up, login, and AI writing forms |
