1. "Check" / "Submit answer" / "Try again" primary action buttons
2. Selected state border on MC option buttons (2px border-primary, pre-submit)
3. Selected left-item highlight in MatchingCard (border-primary, 2px)
4. Paired connection indicator in MatchingCard (border-primary on paired right item, pre-Check)
5. Focus ring on ConjugationTableCard cell inputs (border-b-[3px] border-primary on focus)

Green (`tertiary`) reserved for — and only for:
- `bg-tertiary/10` tint + `border-tertiary` on correct MC option (post-submit)

- `text-tertiary` Check icon (Lucide, 16px) on all correct states

No green anywhere else. No green on the completed toggle state (that uses `primary`). No green on labels, chips, or progress indicators.

Cool tones: none. Warm palette strictly maintained.

## Component Inventory

### 1. MCPracticeCard

Wraps the existing `MCOptionButton` component. Adds prompt display, submit flow, feedback display, and retry logic per D-06/D-07/D-08.

**Anatomy (top to bottom):**
1. Problem prompt — `font-body text-[18px] leading-8 text-on-surface` — the question/instruction text
2. Option list — `role="radiogroup"` wrapping 2–4 `MCOptionButton` instances, `flex flex-col gap-3`
3. Submit button — primary button `"Submit answer"`, enabled only when an option is selected
4. Feedback area (post-submit) — inline correct answer reveal when wrong

**States:**

| State | Description | Visual |
|-------|-------------|--------|
| Default | No option selected | All options: `bg-surface-container-low border-outline` |
| Option selected (pre-submit) | One option highlighted | Selected: `bg-surface-container-low border-primary border-2`; "Submit answer" button enabled |
| Correct (post-submit) | Selected option was correct | Selected option: `bg-tertiary/10 border-tertiary` + Check icon; all options disabled; auto-completes sub-component |
| Incorrect (post-submit) | Selected option was wrong | Wrong option: `bg-error/10 border-error` + X icon; correct option: `bg-tertiary/10 border-tertiary` + Check icon; "Try again" button shown |
| Retry (after incorrect) | Student resets to try again | Options reset to default state (no previous selection shown); "Submit answer" disabled until new selection |
| Completed | Sub-component already complete | All options disabled, correct option shown in correct state; no action button |

**Button labels:**
- Pre-submit: "Submit answer"
- Post-incorrect: "Try again"

**Auto-complete trigger:** correct option submitted → `onComplete(id)` called immediately.

**Retry behavior (D-08):** After seeing incorrect + correct reveal, student presses "Try again". All options reset to default state, `selectedOption` cleared. No wrong answer pre-highlighted on retry.

**Card container:** `bg-surface-container-low border border-outline-variant rounded-[16px] p-6 mt-4`

### 2. FillInPracticeCard

Wraps the existing `FillInInput` component. Adds prompt display, submit flow, feedback display, and retry logic per D-06/D-07/D-08.

**Anatomy (top to bottom):**
1. Problem prompt — `font-body text-[18px] leading-8 text-on-surface`
2. FillInInput — existing component, reused directly (handles focus/correct/incorrect border states internally)
3. Submit button — primary button `"Submit answer"`, enabled when input is non-empty
4. Feedback area (post-submit, incorrect) — correct answer revealed in `font-label text-[13px] text-error` below input

**States:**

| State | Description | Visual |
|-------|-------------|--------|

| Typing | Non-empty input | FillInInput default border; "Submit answer" enabled |

| Retry | After incorrect | Input value cleared; FillInInput resets to default state; "Submit answer" disabled |
| Completed | Sub-component already complete | Input disabled in correct state; no action button |

**Correct answer reveal copy (incorrect state):** "The answer is {correct_answer}" — `font-label text-[13px] text-error`

**Auto-complete trigger:** correct answer submitted → `onComplete(id)` called immediately.

### 3. ConjugationTableCard

NEW component. Two sub-types per D-01:
- **Full table** (6-form): 6-cell grid (je/tu/il/nous/vous/ils), Check button, any submission auto-completes
- **Single blank**: reuses `FillInInput` directly (same as FillInPracticeCard, treated as fill-in sub-type)

This component spec covers the full-table sub-type only. Single blank uses FillInPracticeCard with a single-subject prompt.

**Anatomy (top to bottom):**
1. Problem prompt — `font-body text-[18px] leading-8 text-on-surface` — e.g. "Conjugate _parler_ in the present tense."
2. Conjugation grid — 6 rows × 2 columns (pronoun label | input cell)
3. Check button — primary button `"Check conjugations"`, enabled when at least one cell has content
4. Feedback area (post-submit) — per-cell correct/incorrect states shown, no retry

**Grid layout:**
```
[je  ] [_________]
[tu  ] [_________]
[il  ] [_________]
[nous] [_________]
[vous] [_________]
[ils ] [_________]
```

- Grid: `display grid grid-cols-[auto_1fr] gap-x-3 gap-y-2`
- Pronoun label: `font-label text-[13px] text-on-surface-variant self-center text-right min-w-[36px]`
- Cell input: `min-h-[44px] rounded px-3 py-2 font-body text-[16px] bg-surface-container-low text-on-surface border border-outline outline-none focus:border-b-[3px] focus:border-primary transition-all`

**Cell states:**

| State | Background | Border | Text |
|-------|-----------|--------|------|
| Default | `bg-surface-container-low` | `border-outline` | `text-on-surface` |
| Focused | `bg-surface-container-low` | `border-b-[3px] border-primary` | `text-on-surface` |
| Correct (post-Check) | `bg-tertiary/10` | `border-tertiary border-b-[3px]` | `text-on-surface` |
| Incorrect (post-Check) | `bg-error/10` | `border-error border-b-[3px]` | `text-on-surface` |
| Disabled (post-Check) | `bg-surface-container` | `border-outline-variant` | `text-on-surface-variant opacity-60` (empty cells only) |

**Post-Check feedback:**
- Each cell independently shows correct (tertiary) or incorrect (error) state
- Correct form displayed below each incorrect cell: `font-label text-[13px] text-error` — "Answer: {correct_form}"
- No retry — single submission per D-09
- Sub-component auto-completes immediately on Check click (regardless of score) per D-06

**Auto-complete trigger:** "Check conjugations" clicked → score cells → call `onComplete(id)` immediately.

**Tab navigation:** Tab key moves focus cell-to-cell in order (je → tu → il → nous → vous → ils). Enter in last cell triggers Check.

**Card container:** `bg-surface-container-low border border-outline-variant rounded-[16px] p-6 mt-4`

### 4. MatchingCard

NEW component. Click-to-pair interaction per D-03. All-at-once feedback after Check per D-04. Auto-completes regardless of score per D-05.

**Anatomy (top to bottom):**
1. Problem prompt — `font-body text-[18px] leading-8 text-on-surface` — e.g. "Match each word to its translation."
2. Two-column pairing area — left column (source items) | right column (target items)
3. Check button — primary button `"Check matches"`, enabled when all left items are paired
4. Feedback area (post-Check) — color overlay on each pair, no retry

**Two-column layout:**
- Container: `grid grid-cols-2 gap-4`
- Left column: source items (French words or infinitives) — `flex flex-col gap-2`
- Right column: target items (translations or conjugated forms) — `flex flex-col gap-2`
- Items in right column are shuffled at render time (not in same order as left)

**Item button anatomy:**
- Container: `min-h-[44px] w-full flex items-center px-4 py-3 rounded-[8px] border text-left font-body text-[16px] transition-colors`

**Left item states:**

| State | Background | Border | Text |
|-------|-----------|--------|------|
| Default (unselected, unpaired) | `bg-surface-container-low` | `border-outline` | `text-on-surface` |
| Selected (awaiting right pick) | `bg-surface-container` | `border-primary border-2` | `text-on-surface` |
| Paired (pre-Check) | `bg-surface-container-high` | `border-primary` | `text-on-surface` |
| Correct (post-Check) | `bg-tertiary/10` | `border-tertiary` | `text-on-surface` |
| Incorrect (post-Check) | `bg-error/10` | `border-error` | `text-on-surface` |
| Disabled | Same as paired/correct/incorrect state | — | `cursor-default` |

**Right item states:**

| State | Background | Border | Text |
|-------|-----------|--------|------|
| Default (unpaired) | `bg-surface-container-low` | `border-outline` | `text-on-surface` |
| Paired (pre-Check) | `bg-surface-container-high` | `border-primary` | `text-on-surface` |
| Correct (post-Check) | `bg-tertiary/10` | `border-tertiary` | `text-on-surface` |
| Incorrect (post-Check) | `bg-error/10` | `border-error` | `text-on-surface` |

**Click-to-pair interaction flow (D-03):**
1. Student clicks a left item → it enters "selected" state (`border-primary border-2`)
2. Student clicks a right item → the pair is formed; both enter "paired" state (`bg-surface-container-high border-primary`)
3. Student can click a left item that is already paired → it re-enters "selected" state, its existing right-item partner reverts to unpaired; student can then pick a new right item
4. Student can click an unpaired left item while another left item is already selected → selection transfers
5. Once all left items have a right-item partner → "Check matches" button enables
6. Student clicks "Check matches" → all pairs evaluated at once → correct/incorrect states applied → `onComplete(id)` called immediately

**Correct answer on incorrect pairs (post-Check):**
- Below the two-column grid, a single `font-label text-[13px] text-error` line per incorrect pair:
  `"{left_item}" pairs with "{correct_right_item}"`
- Shown only for incorrect pairs, in order of left column

**Auto-complete trigger:** "Check matches" clicked → `onComplete(id)` called immediately (D-05).

**Problem size:** 4–6 pairs. Column layout stable at this range on 720px content width.

**Card container:** `bg-surface-container-low border border-outline-variant rounded-[16px] p-6 mt-4`

### SubComponentItem — kind='practice' integration

**Current behavior (Phase 3):** `kind='practice'` renders the title row (toggle + title + chip) but no content below — the `content && ...` block is only reached for explainers that have markdown.

**Phase 5 change:** Add a new branch below the toggle row that renders the appropriate problem component. The toggle button for practice items is hidden (not clickable) — completion is driven by the problem result, not manual click.

**Integration anatomy:**

```
<div className="py-2">
  <div className="flex items-center gap-3">
    {/* Toggle button: hidden for practice kind — replaced by a non-interactive spacer */}
    {/* Title + kind chip row */}
  </div>
  
  {/* Practice problem panel — renders below title row, indented 60px to align with content */}
  {kind === 'practice' && problemData && (
    <div className="mt-4 sm:ml-[60px]">
      {/* PracticeCardRouter selects MCPracticeCard | FillInPracticeCard |
          ConjugationTableCard | MatchingCard based on problemData.type */}
    </div>
  )}
</div>
```

**Toggle button behavior for practice kind:**
- The 48px circle button is replaced by a visual-only indicator (same size spacer) when `kind === 'practice'`
- When `isCompleted`: spacer shows `bg-primary border-primary` filled checkmark (same visual as other kinds)
- When incomplete: spacer shows `bg-surface-container-high border-outline-variant` muted circle (no dash icon — not user-clickable)
- `aria-label` on the spacer: `{title} — answer to complete` (describes how to complete without implying clickability)
- The spacer is `role="presentation"` and not focusable (`tabIndex={-1}`)

**onComplete callback:** Each problem card calls `onComplete(id)` — the same Server Action prop already threaded through SubComponentItem. No prop changes to SubComponentItem's external API; internal branching on `kind`.

**Problem data shape (design contract only — data model is planner/researcher scope):**
Each practice sub-component requires a `type` field to route to the correct card:
- `'mc'` → MCPracticeCard
- `'fill-in'` → FillInPracticeCard
- `'conjugation-table'` → ConjugationTableCard (full 6-form)
- `'conjugation-single'` → FillInPracticeCard (reuses fill-in card with single subject prompt)
- `'matching'` → MatchingCard

## Interaction Contracts

### MC Practice flow
1. Problem card renders with prompt and 2–4 options in default state
2. Student selects an option → option border highlights (primary, 2px); "Submit answer" enables
3. Student clicks "Submit answer" → option states revealed (correct=tertiary, others=outline-variant at opacity-60)
4. If correct → `onComplete(id)` fires; card shows completed state; no action button
5. If incorrect → wrong option shows error state; correct option shows tertiary state; "Try again" appears
6. Student clicks "Try again" → all options reset to default state, no selection; "Submit answer" disabled again; student can pick a new answer

### Fill-in Practice flow
1. Problem card renders with prompt and empty FillInInput
2. Student types → "Submit answer" enables when non-empty
3. Student presses Enter or clicks "Submit answer" → answer compared (accent-insensitive)
4. If correct (exact or accent-insensitive) → FillInInput shows correct state; `onComplete(id)` fires; accent note shown if applicable; no action button
5. If incorrect → FillInInput shows error state; correct answer shown below in `text-error font-label text-[13px]`; "Try again" appears
6. Student clicks "Try again" → input value cleared; FillInInput resets to default; "Submit answer" disabled

### Conjugation table flow
1. Card renders with prompt and 6 empty cell inputs
2. Student fills cells (Tab to advance, Enter in last cell to submit)
3. "Check conjugations" enables when at least one cell has content
4. Student clicks "Check conjugations" → each cell evaluated independently → correct/incorrect states applied per cell → incorrect cells show correct form below → `onComplete(id)` fires immediately
5. No retry — all cells disabled after Check

### Matching flow
1. Card renders with prompt and two shuffled columns
2. Student clicks a left item → selected state (primary border, 2px)
3. Student clicks a right item → pair formed (both show paired state, primary border)
4. Student can re-pair by clicking any already-paired left item
5. "Check matches" enables when all left items have a right-item partner
6. Student clicks "Check matches" → all pairs evaluated at once → correct (tertiary) / incorrect (error) states applied → incorrect pairs listed below → `onComplete(id)` fires immediately
7. No retry

### Keyboard navigation
- MCPracticeCard: Tab through options, Space/Enter to select, Tab to "Submit answer", Enter to submit
- FillInPracticeCard: Focus input on mount, Enter to submit
- ConjugationTableCard: Tab through cells in order, Enter on last cell triggers Check
- MatchingCard: Tab through left items, Space/Enter to select left item, Tab through right items, Space/Enter to pair

### Focus management on retry (MC and fill-in)
After clicking "Try again": focus moves to the first option (MC) or the input field (fill-in). Student does not need to re-navigate from the top.

### Completed state (all types)
When `isCompleted` prop is true on mount: card renders in its completed/correct visual state with all inputs/options disabled. No action button shown. This handles page reload after prior completion.

## Accessibility Contract

- All interactive elements: minimum 44px height touch target
- MCPracticeCard options: `role="radio"` within `role="radiogroup"` with `aria-label="Answer options"` on group; `aria-checked` on each option button (inherited from MCOptionButton)
- Correct/incorrect icons: `aria-hidden="true"` on Lucide icons; result conveyed via text feedback below
- ConjugationTableCard cells: each `<input>` has `aria-label="{pronoun} form"` (e.g. `aria-label="je form"`)
- ConjugationTableCard: grid region has `aria-label="Conjugation table"` on the wrapping div
- MatchingCard left items: `role="button" aria-pressed={isPaired}` with `aria-label="{text} — {unpaired | selected | paired with {partner}}"`
- MatchingCard right items: `role="button" aria-label="{text} — {available | paired with {partner}}"`
- Correct/incorrect states (conjugation, matching): `role="status"` region below each card announces result count: "3 of 6 correct" (polite live region)
- "Try again" button: `aria-label="Try again — {title}"` to distinguish if multiple problems on page
- Keyboard: Tab/Space/Enter navigation throughout; no mouse-only interactions
- Focus on retry: focus returned to first option or input after "Try again"
- Error messages: `role="alert"` for unexpected errors
- Completed sub-component spacer: `role="presentation" tabIndex={-1}` — not in tab order
